2. REST Nədir? (Representational State Transfer)

    - REST, proqramların bir-biri ilə danışmaq qaydasıdır. 



Əsas prinsiplərini başa düşmək üçün misallar:

    1. Stateless (yadda saxlamır)
        - Hər sorğu (istək) müstəqildir.

        - Məsələn: Banka gedib balansını soruşursan → “Balansın 100 manatdır” deyirlər.
        Sonra yenidən soruşsan, bank əvvəlkini xatırlamır, yenə 100 manat deyir.
        Yəni keçmişi unudur, hər dəfə sıfırdan baxır.




    2. HTTP istifadə edir
        - İnternetdə sayt açanda necə http://... yazırsan, elə API-lər də bu üsulu istifadə edir.

        - Məsələn:
            - GET – məlumatı oxu (kitabı gör).
            - POST – məlumat əlavə et (yeni kitab əlavə et).
            - PUT/PATCH – mövcud məlumatı dəyiş (kitabın qiymətini dəyiş).
            - DELETE – məlumatı sil (kitabı sil).




    3. Resurs anlayışı
        - Serverdə hər şey bir resurs kimi qəbul olunur.
        - Məsələn, kitabxanada “kitab” bir resursdur. Onu görmək, əlavə etmək, dəyişmək və ya silmək olar.




REST-in üstünlükləri:
    - Sadədir və başa düşüləndir.
    - Hər proqramlaşdırma dilində işləyir (Python, Java, JavaScript və s.).
    - Çox sürətlidir və internet üçün uyğundur.


Həyatdakı misal:
    - Telefonda hava proqramını açırsan.
    - Sual verirsən: “Bakıda hava necədir?”
    - Server cavab verir: “20°, günəşli.”
    - Sabah yenidən soruşsan, əvvəlki cavabı xatırlamır, yenidən cavab verir.



Yəni REST API sadəcə "serverdən məlumat soruşma və cavab alma qaydasıdır".
