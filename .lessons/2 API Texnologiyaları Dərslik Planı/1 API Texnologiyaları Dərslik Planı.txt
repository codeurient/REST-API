API texnologiyaları dərs planı:

    - Bu plan həm nəzəri, həm də praktik hissələrdən ibarətdir və addım-addım öyrədici şəkildə qurulub.




------------------------------------------------------------------------------------------------------------------------


API Texnologiyaları Dərslik Planı

I) Giriş – API nədir?

- API anlayışı (Application Programming Interface).

- API növləri:
    - Local API
    - Web API
    - Library API

- Niyə API öyrənmək vacibdir? (real dünya tətbiqləri, inteqrasiya, mikrosistemlər).


------------------------------------------------------------------------------------------------------------------------


II) HTTP Əsasları (bütün API-lərin təməli)

Nəzəri:
    - HTTP nədir?
    - HTTP metodları: GET, POST, PUT, DELETE, PATCH.
    - Status kodları: 200, 201, 400, 401, 404, 500.
    - Headers və Body.

Praktika:
    - Sadə curl sorğuları.
    - Postman / Insomnia ilə test etmək.


------------------------------------------------------------------------------------------------------------------------


III) REST API və RESTful üslub

Nəzəri:
- REST prinsipləri (Stateless, Cacheable, Uniform Interface).
- REST və RESTful fərqi.
- JSON əsaslı məlumat ötürülməsi.

Praktika:
    - Todo API nümunəsi (CRUD əməliyyatları).
    - Frontend-dən (React/Vue) REST API-yə sorğu göndərmək.


------------------------------------------------------------------------------------------------------------------------


IV) GraphQL (Müasir API Yanaşması)

Nəzəri:
    - REST və GraphQL müqayisəsi (overfetching / underfetching).
    - Schema, Query, Mutation anlayışı.

Praktika:
    - Sadə GraphQL server qurmaq (Apollo Server / Yoga).
    - Sorğu: yalnız name və email alan istifadəçi siyahısı.



------------------------------------------------------------------------------------------------------------------------


V) Real-time API-lər

a) WebSockets
    - Əlaqənin “iki tərəfli” olması.
    - REST ilə fərqi.
    - Praktika: Sadə chat app.

b) SSE (Server-Sent Events)
    - Bir tərəfli real-time məlumat ötürülməsi.
    - Praktika: Canlı “clock” və ya “stock price” stream.


------------------------------------------------------------------------------------------------------------------------


VI) SOAP (Köhnə, amma vacib)

Nəzəri:
    - XML əsaslı API.
    - REST ilə müqayisə (çətinliklər, standartlar, enterprise istifadə).

Praktika:
    - Sadə SOAP xidmətini çağırmaq (məsələn, “currency conversion” xidmətindən istifadə).



------------------------------------------------------------------------------------------------------------------------


VII) RPC (Remote Procedure Call)

Nəzəri:
    - RPC anlayışı: funksiya çağırışı kimi API.
    - JSON-RPC, XML-RPC, gRPC müqayisəsi.

Praktika:
    - İki mikroservis arasında sorğu göndərmək.


------------------------------------------------------------------------------------------------------------------------


VIII) Mesajlaşma əsaslı API-lər

a) MQTT (IoT üçün)
    - Sensor məlumatı ötürmək üçün ideal.
    - Praktika: temperature mövzusu ilə publish-subscribe nümunəsi.

b) AMQP (RabbitMQ ilə)
    - Queue əsaslı sistemlər (mesaj broker).
    - Praktika: Producer və Consumer nümunəsi.


------------------------------------------------------------------------------------------------------------------------


IX) Digər Texnologiyalar (qısa təqdimat)
    - HATEOAS – REST-in genişləndirilməsi.
    - OData – Microsoft-un açıq data protokolu.
    - Falcor – Netflix-in alternativi GraphQL-ə.
    - Thrift və Avro – Big Data və mikroservis inteqrasiyası üçün.
    - CORBA – Tarixi sistemlərdə istifadə olunub.

(İzah + qısa demo, dərindən girməyə ehtiyac yoxdur).


------------------------------------------------------------------------------------------------------------------------


X) Webhooks

Nəzəri:
    - Event-based yanaşma (məsələn, “payment success” → webhook çağırılır).

Praktika:
    - Stripe və ya PayPal sandbox ilə webhook dinləyici qurmaq.


------------------------------------------------------------------------------------------------------------------------



XI. Təkrar və Layihə

Böyük layihə:

    - “E-commerce mini API sistemi”
        - REST API → məhsullar və istifadəçilər
        - GraphQL → məhsul sorğuları
        - WebSockets → canlı chat və ya sifariş statusu
        - Webhook → ödəniş tamamlandıqda bildiriş

(əlavə olaraq MQTT və ya gRPC də inteqrasiya edilə bilər). Bunu ola bilsin etmədik.
