Bunlar API-ləri tərtib etmək, sənədləşdirmək, spesifikasiya yazmaq və testlərini avtomatlaşdırmaq üçündür:



- Swagger (OpenAPI Specification)
- Redoc
- Stoplight
- Apicurio
- Dredd
- RAML
- API Blueprint
- RapidAPI




Swagger
    - Əslində OpenAPI Specification (OAS) standartını dəstəkləyən alətlər dəstinə verilən addır.
    - Məqsədi: API-ləri sənədləşdirmək və test etmək.
    - Ən məşhur komponenti → Swagger UI → Brauzerdə API sənədlərini interaktiv görürsən və birbaşa test edə bilirsən.
    - Məsələn: https://example.com/api-docs → burda sənədləşmə + test imkanı bir yerdə olur.
    👉 Daha çox backend developer-lər və komandalar üçün yararlıdır (çünki həm sənədləşmə, həm test, həm də standartlaşdırma verir).







Redoc
    - Açıq mənbəli (open-source).
    - OpenAPI sənədlərini çox gözəl və oxunaqlı interfeysə çevirir.
    - Daha çox sənədləşmə üçün istifadə olunur (interaktiv test Swagger qədər güclü deyil).











Stoplight
    - API dizayn, sənədləşmə və test üçün “hamısı bir yerdə” platformadır.
    - Vizual interfeysi var → texniki olmayan insanlar da başa düşə bilir.
    - OpenAPI dəstəkləyir.











Apicurio
    - Açıq mənbəli API dizayn aləti.
    - OpenAPI və AsyncAPI ilə işləyə bilir.
    - Komanda ilə işləmək üçün uyğundur.













Dredd
    - API sənədləşməsini yoxlamaq üçün test alətidir.
    - OpenAPI sənədlərinə baxır və onların həqiqətən server cavabları ilə uyğun olub-olmadığını test edir.
    - Daha çox QA və test üçün.












RAML (RESTful API Modeling Language)
    - Swagger kimi API-ləri təsvir etmək üçün başqa bir standartdır.
    - RAML ilə yazılan sənədləri oxumaq və test etmək üçün fərqli alətlər var.










API Blueprint
    - Markdown əsaslı API sənədləşmə dili.
    - Sadə yazılır, sonra alətlər vasitəsilə gözəl sənədlərə çevrilir.








RapidAPI
    - API bazarı + test + sənədləşmə birləşmiş formada.
    - OpenAPI sənədlərini import edə bilir.
    - Əlavə olaraq API-ləri paylaşmaq olur.
