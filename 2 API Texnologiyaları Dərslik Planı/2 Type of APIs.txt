1. Local API

Tərifi: Local API (yerli API) eyni cihaz və ya sistem daxilində işləyən proqramlar arasında məlumat mübadiləsini 
təmin edən interfeyslərdir. Bu API-lər adətən bir kompüterin və ya proqramın daxili komponentləri ilə işləyir və 
şəbəkə bağlantısına ehtiyac duymur.



Xüsusiyyətləri:

    - Yerli işləmə: Local API-lər eyni maşında və ya proqram mühitində çalışır. Məsələn, əməliyyat sisteminin funksiyalarına (Windows API, POSIX) 
      və ya cihazın drayverlərinə daxil olmaq üçün istifadə olunur.
    - Sürət: Şəbəkə bağlantısı tələb olunmadığından daha sürətlidir.
    - Təhlükəsizlik: Məlumatlar şəbəkəyə çıxmır, buna görə daxili sistemlərdə daha təhlükəsizdir.
    - Məhdudiyyətlər: Yalnız yerli mühitdə işləyir, uzaq sistemlərlə inteqrasiya üçün uyğun deyil.

Nümunələr:

    - Windows API: Windows əməliyyat sistemində fayl idarəetməsi, qrafik interfeys və ya sistem resurslarına daxil olmaq üçün.
    - POSIX API: Unix/Linux sistemlərində proseslər və fayl sistemləri ilə işləmək üçün.
    - Java Runtime Environment (JRE) daxilindəki siniflər və metodlar.

İstifadə sahələri:

    - Əməliyyat sistemləri ilə işləyən proqramlar.
    - Cihaz drayverləri ilə inteqrasiya.
    - Performansın vacib olduğu yerli tətbiqlər.

2. Web API


Tərifi: Web API internet və ya şəbəkə üzərindən fərqli sistemlər və ya tətbiqlər arasında məlumat mübadiləsini təmin edən API növüdür. 
Bu API-lər adətən HTTP/HTTPS protokolları vasitəsilə işləyir və müştəri-server arxitekturasına əsaslanır.

Xüsusiyyətləri:

    - Şəbəkə əsaslı: Web API-lər internet üzərindən sorğular (GET, POST, PUT, DELETE) qəbul edir və cavablar (JSON, XML) göndərir.
    - Platformadan asılı olmayaraq: Fərqli proqramlaşdırma dilləri, platformalar və cihazlar tərəfindən istifadə edilə bilər.
    - Standartlaşdırma: REST (Representational State Transfer), SOAP və ya GraphQL kimi standartlara əsaslanır.
    - Təhlükəsizlik: OAuth, API açarları və ya JWT (JSON Web Token) kimi autentifikasiya metodları ilə qorunur.
    - Məhdudiyyətlər: Şəbəkə gecikmələri və internet bağlantısına ehtiyac səbəbindən performans yerli API-lərdən aşağı ola bilər.

Nümunələr:

    - Google Maps API: Xəritə xidmətləri və geolokasiya məlumatları üçün.
    - Twitter (X) API: Sosial media məzmununa daxil olmaq və ya paylaşım etmək üçün.
    - PayPal API: Ödəniş sistemləri ilə inteqrasiya üçün.
    - RESTful API-lər: JSON formatında məlumat mübadiləsi üçün geniş istifadə olunur.

İstifadə sahələri:

    - Veb tətbiqləri və mobil proqramlar.
    - Bulud xidmətləri ilə inteqrasiya (AWS, Azure).
    - Üçüncü tərəf servisləri ilə məlumat mübadiləsi.

3. Library API

Tərifi: Library API bir proqramlaşdırma kitabxanasının və ya modulu daxilində təqdim olunan funksiyalar, siniflər və ya metodlar 
toplusudur. Bu API-lər proqramçılara müəyyən funksionallığı təkrar yazmadan istifadə etməyə imkan verir.


Xüsusiyyətləri:

    - Modul əsaslı: Library API-lər adətən müəyyən bir proqramlaşdırma dilində yazılmış kitabxanalar (libraries) şəklində olur.
    - Yerli və ya quraşdırılmış: Kitabxana proqramın bir hissəsi kimi quraşdırılır və yerli olaraq işləyir.
    - Asılılıq: Proqramın işləməsi üçün kitabxananın düzgün quraşdırılması tələb olunur.
    - Məhdudiyyətlər: Library API-lər yalnız müəyyən bir proqramlaşdırma dili və ya platforma ilə məhdudlaşa bilər.

Nümunələr:

    - Python-un NumPy və ya Pandas kitabxanaları: Məlumat analizi və riyazi əməliyyatlar üçün.
    - JavaScript-in React kitabxanası: İnterfeys qurmaq üçün.
    - C++ STL (Standard Template Library): Verilənlər strukturları və alqoritmlər üçün.

İstifadə sahələri:

    - Təkrar istifadə oluna bilən kod yazmaq.
    - Mürəkkəb funksiyaları sadələşdirmək.
    - Proqramlaşdırma dillərinin standart və ya xarici kitabxanalarından istifadə.
